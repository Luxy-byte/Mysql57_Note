# MySAM 对比 InnoDB

**常用数据库引擎：**

|            | MySAM  | INNODB                |
| ---------- | ------ | --------------------- |
| 事物支持   | 不支持 | 支持                  |
| 数据行锁定 | 不支持 | 支持                  |
| 外键约束   | 不支持 | 支持                  |
| 全文索引   | 支持   | 不支持                |
| 表空间大小 | 较小   | 较大， 约为 2倍 MySAM |

- MYISAM：节约空间，速度较快
- INNODB：安全性高，事物的处理，多表多用户操作



> 在物理空间存在的位置

所有的数据库文件都存在data目录下，本质是 文件存储

- INNODB 在数据库表中只有一个 *.frm 文件，以及上级目录下的 ibdata1 文件
- MYISAM 对应文件：
  - *.frm  表结构的定义文件
  - *.MYD  数据文件(data)
  - *.MYI   索引文件(index)



> 数据库表的字符集编码

```sql
CHARSET = UTF8
不设置的话，mysql默认的字符集编码 ：Latin1（不支持中文）

在my.ini中配置默认的编码：
character-set-server = utf8
```
